# 终端早餐店 - 操作说明

## 快速开始

### 启动服务

```bash
cd /home/exam
./start.sh start
```

### 连接终端

使用 `nc` (netcat) 或 telnet 连接：

```bash
# 叫号牌端 (只读)
node client.js 8024

# 顾客下单端
node client.js 8025

# 店主管理端
node client.js 8026
```

### 停止服务

```bash
./start.sh stop
```

## 顾客下单端命令

| 命令 | 说明 | 示例 |
|------|------|------|
| `help` / `?` | 显示帮助 | `help` |
| `status` | 查看系统状态 | `status` |
| `menu` | 查看菜单 | `menu` |
| `name <昵称>` | 设置昵称 | `name 张三` |
| `order <菜品>` | 下单 | `order 1,2 3,1` |
| `myorder` | 查看我的订单 | `myorder` |
| `pay <订单号>` | 支付订单 | `pay ORD123456` |
| `cancel <订单号>` | 取消订单 | `cancel ORD123456` |
| `quit` / `exit` | 退出 | `quit` |

### 下单格式

```
order <物品ID>,<数量> [<物品ID>,<数量> ...]
```

示例：
- `order 1` - 点1份豆浆
- `order 1,2` - 点2份豆浆
- `order 1,2 3,1` - 点2份豆浆 + 1份包子

## 店主管理端命令

| 命令 | 说明 | 示例 |
|------|------|------|
| `help` / `?` | 显示帮助 | `help` |
| `status` | 查看系统状态 | `status` |
| `list` | 查看所有订单 | `list` |
| `pending` | 查看待支付订单 | `pending` |
| `paid` | 查看已支付订单 | `paid` |
| `cooking` | 查看制作中订单 | `cooking` |
| `call <号码>` | 设置当前叫号 | `call 1001` |
| `next` | 叫下一个号 | `next` |
| `update <订单号> <状态>` | 更新订单状态 | `update ORD123 cooking` |
| `batch <状态>` | 批量更新已支付订单 | `batch cooking` |
| `refresh` | 刷新订单列表 | `refresh` |

### 状态说明

| 状态 | 说明 |
|------|------|
| `pending` | 待支付 |
| `paid` | 已支付 |
| `cooking` | 制作中 |
| `completed` | 已完成 |

### 批量操作

```bash
# 将所有已支付订单设为制作中
batch cooking

# 将所有制作中订单设为已完成
batch completed
```

## 叫号牌端

叫号牌端为**只读**模式，自动显示：
- 当前叫号
- 等待队列
- 订单状态

每3秒自动刷新，也可输入 `refresh` 手动刷新。

## 完整业务流程

### 顾客端流程

```
1. 连接顾客端 (node client.js 8025)
2. 设置昵称 (可选): name <你的名字>
3. 查看菜单: menu
4. 下单: order <菜品>
5. 支付: pay <订单号>
6. 等待叫号
```

### 店主端流程

```
1. 连接店主端 (node client.js 8026)
2. 查看订单: list
3. 查看待支付订单: paid
4. 叫号: next
5. 开始制作: update <订单号> cooking
6. 完成制作: update <订单号> completed
```

---

## 状态流转规则

```
PENDING (待支付)
    ↓ 支付
PAID (已支付)
    ↓ 开始制作
COOKING (制作中)
    ↓ 制作完成
COMPLETED (已完成)

PENDING → CANCELLED (取消)
```

**注意**: 非法状态跳转会被系统拒绝！

## 常见问题

### Q: 连接失败怎么办？
A: 确保服务已启动 `./start.sh status`，检查端口是否被占用。

### Q: 订单号大小写？
A: 系统会自动将输入的订单号转换为大写，无需担心大小写。

### Q: 如何查看日志？
A: `./start.sh log` 或 `tail -f /tmp/breakfast_server.log`

### Q: 数据会丢失吗？
A: 不会，系统每5分钟自动保存数据到 `/home/exam/data/store.json`
