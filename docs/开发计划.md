# 多端 TUI 早餐店系统 - 开发计划

**技术栈**：Node.js + 自建 TCP 协议 + ANSI TUI

---

## 里程碑一：基础架构搭建 (MVP)

### 任务 1.1：项目初始化
- [ ] 初始化 Node.js 项目 (`npm init`)
- [ ] 安装必要依赖 (无外部依赖，纯 Node.js 内置模块)
- [ ] 创建目录结构

```
/src
  /server        # 服务端
    index.js     # 入口
    state.js     # 状态管理
    protocol.js  # 协议解析
  /client        # 客户端
    index.js     # 入口
    terminal.js  # TUI 渲染
  /shared        # 共享代码
    menu.js      # 菜单数据
    status.js    # 状态枚举
  /test          # 测试脚本
```

### 任务 1.2：TCP 服务器框架
- [ ] 实现基础 TCP 服务器，监听 8024/8025/8026 端口
- [ ] 实现连接管理（会话创建/销毁）
- [ ] 实现简单的命令路由

### 任务 1.3：核心状态管理
- [ ] 定义订单状态枚举 (`PENDING`, `PAID`, `COOKING`, `COMPLETED`, `CANCELLED`)
- [ ] 实现内存状态存储（订单列表、当前叫号）
- [ ] 实现状态变更方法（含合法性校验）
- [ ] 实现事件广播机制

### 任务 1.4：客户端 TUI 框架
- [ ] 实现 ANSI 转义序列封装
- [ ] 实现清屏、光标控制、颜色输出
- [ ] 实现基础输入处理

## 阶段验收标准

| 阶段 | 验收条件 |
|------|----------|
| 里程碑一 | TCP 服务器启动成功，三个端口可连接，状态存储工作正常 |
| 里程碑二 | 三个终端可独立完成各自功能，界面可读 |
| 里程碑三 | 状态变更可实时同步到所有终端 |
| 里程碑四 | 所有测试用例通过，无明显 bug |
| 里程碑五 | 一键启动正常，文档齐全 |

## 优先级排序

**P0（必须完成）**：
1. TCP 服务器框架
2. 状态管理与状态机
3. 三个终端基本功能
4. 状态实时同步
5. 状态流转合法性

**P1（应该完成）**：
1. 并发控制
2. 断连处理

1. 持久化存储

## 启动命令预览

```bash
# 启动服务端
node src/server/index.js

# 客户端连接（使用 netcat 或 telnet）
nc localhost 8024  # 叫号牌端
nc localhost 8025  # 顾客下单端
nc localhost 8026  # 店主管理端
```
