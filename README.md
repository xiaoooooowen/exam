# 终端早餐店

## 题目目标

实现一个多端 TUI 系统，模拟早餐店从下单到制作到支付确认的完整流程。系统包含三个终端入口：
- 叫号牌端（只读）
- 顾客下单端（可操作）
- 店主管理端（可操作）

## 端口与角色要求

1. 叫号牌端：`ssh localhost:8024`
- 只读
- 自动刷新当前叫号、制作状态、支付状态

2. 下单端：`ssh localhost:8025`
- 可多人连接
- 每个连接获得号码牌
- 可下单并展示支付二维码（可 Mock）

3. 店主端：`ssh localhost:8026`
- 可查看订单列表
- 可修改订单状态（待支付/已支付/制作中/等）

## 技术约束（必读）
- 可使用自建 SSH/Telnet 或其他终端协议方案，但需说明架构。

## 功能要求（必做）
- 多端共享同一订单状态，状态变更实时可见。
- 订单状态流转必须合法（禁止非法跳转）。
- 顾客端支持基础菜单浏览与下单。
- 店主端支持按订单号更新状态。

## 测试与验收要求（必做）

至少执行以下测试并记录结果：

1. 多连接测试
2. 状态流转测试
3. 断连恢复测试
4. 并发一致性测试

## 交付物

- 服务器与客户端源码
- 一键启动脚本
- 终端操作说明（快捷键/命令）
- 测试脚本与结果
- 架构说明（协议选型、状态同步方案）

## 加分项
- 持久化存储
- 基础权限控制
- 高可读的 TUI 交互设计
