### 1. 多连接测试

**测试目标**: 验证三个终端可以同时连接并正常工作

**测试步骤**:
1. 同时连接到 8024 (叫号牌端)、8025 (顾客端)、8026 (店主端)
2. 各终端发送命令验证响应

**测试命令**:
```bash
node test/multi-connection-test.js
```

**预期结果**:
- 叫号牌端: 显示叫号界面
- 顾客端: 显示欢迎信息和菜单
- 店主端: 显示订单列表

**实际结果**: ✅ 通过

### 2. 状态流转测试

**测试目标**: 验证订单状态机正确工作，非法跳转被拒绝

**测试步骤**:
1. 顾客下单
2. 支付订单 (PENDING → PAID)
3. 尝试非法跳转 (PAID → COMPLETED，应被拒绝)
4. 正确流程 (PAID → COOKING → COMPLETED)

**测试结果**:

| 操作 | 预期状态 | 实际结果 |
|------|----------|----------|
| 下单 | PENDING | ✅ 正确 |
| 支付 | PAID | ✅ 正确 |
| 非法跳转 PAID→COMPLETED | 被拒绝 | ✅ 正确拒绝 |
| COOKING | COOKING | ✅ 正确 |
| COMPLETED | COMPLETED | ✅ 正确 |

**状态机校验**: ✅ 通过

### 3. 断连恢复测试
