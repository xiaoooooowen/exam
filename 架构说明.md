# 多端 TUI 早餐店系统 - 架构说明

## 一、系统架构

### 1.1 整体架构

```
┌─────────────────────────────────────────────────────────┐
│                    TCP Client                           │
│  (nc/telnet/自定义客户端)                               │
└──────────────────┬──────────────────────────────────────┘
                   │ TCP (自定义文本协议)
┌──────────────────▼──────────────────────────────────────┐
│                   TCP Server                            │
│  ┌──────────────┬──────────────┬──────────────┐        │
│  │ 叫号牌端     │ 顾客下单端   │ 店主管理端   │        │
│  │ (8024)       │ (8025)       │ (8026)       │        │
│  └──────┬───────┴──────┬───────┴──────┬───────┘        │
│         │              │              │                 │
│  ┌──────▼──────────────▼──────────────▼───────┐        │
│  │           协议处理层 (protocol.js)          │        │
│  │  - 命令解析                                  │        │
│  │  - 业务逻辑                                  │        │
│  │  - TUI 渲染                                  │        │
│  └─────────────────────┬───────────────────────┘        │
│                        │                                 │
│  ┌─────────────────────▼───────────────────────┐        │
│  │           状态管理层 (state.js)              │        │
│  │  - 订单管理                                  │        │
│  │  - 状态机                                    │        │
│  │  - 事件广播                                  │        │
│  │  - 持久化存储                                │        │
│  └──────────────────────────────────────────────┘        │
└─────────────────────────────────────────────────────────┘
```

### 1.2 技术选型

| 组件 | 技术 | 理由 |
|------|------|------|
| 后端核心 | Node.js | 事件驱动、高并发、I/O 友好 |
| 网络通信 | Node.js net | 内置 TCP 模块，无需额外依赖 |
| 状态存储 | 内存 Map + JSON 文件 | 简单可靠、支持持久化 |
| TUI 渲染 | ANSI 转义序列 | 跨平台终端兼容 |
| 客户端 | nc/telnet | 无需开发，使用标准工具 |

## 二、核心模块

### 2.1 服务器入口 (src/server/index.js)

```
职责：
- 启动三个 TCP 服务器 (8024/8025/8026)
- 管理客户端连接
- 初始化事件监听
- 广播消息到各终端

关键函数：
- createServer(terminalType) - 创建指定类型的服务器
- broadcastToType() - 广播消息到指定终端类型
- initEventListeners() - 初始化事件监听
```

### 2.2 状态管理 (src/server/state.js)

```
职责：
- 订单的增删改查
- 状态机逻辑
- 事件广播
- 数据持久化

核心数据结构：
- orders: Map<OrderId, Order>
- currentCallNumber: number
- listeners: Set<Function>

核心函数：
- createOrder() - 创建订单
- updateOrderStatus() - 更新订单状态（含状态机校验）
- getOrders() - 获取订单列表
- addListener() - 添加事件监听
- saveData() / loadData() - 持久化
```

### 2.3 协议处理 (src/server/protocol.js)

```
职责：
- 解析用户命令
- 分发到对应处理函数
- 渲染 TUI 输出

命令路由：
- handleCallBoardCommand() - 叫号牌端命令
- handleCustomerCommand() - 顾客端命令
- handleShopOwnerCommand() - 店主端命令

核心函数：
- processCommand() - 主命令处理器
- renderCallBoard() - 叫号牌界面渲染
- renderMenu() - 菜单渲染
- renderAllOrders() - 订单列表渲染
```

## 三、状态同步方案

### 3.1 事件驱动架构

```
订单创建 ──→ broadcast('order:created')
                  │
                  ├─→ 叫号牌端: 刷新显示
                  └─→ 店主端: 新订单通知

订单状态变更 ──→ broadcast('order:status_changed')
                  │
                  ├─→ 叫号牌端: 刷新显示
                  └─→ 店主端: 状态变更通知

叫号变更 ──→ broadcast('call:changed')
                │
                └─→ 叫号牌端: 刷新显示
```

### 3.2 同步机制

| 事件 | 触发时机 | 广播范围 |
|------|----------|----------|
| order:created | 新订单创建 | 店主端 |
| order:status_changed | 状态变更 | 叫号牌端 + 店主端 |
| call:changed | 叫号更新 | 叫号牌端 |

## 四、订单状态机

### 4.1 状态定义

```javascript
const OrderStatus = {
  PENDING: 'PENDING',     // 待支付
  PAID: 'PAID',           // 已支付
  COOKING: 'COOKING',     // 制作中
  COMPLETED: 'COMPLETED', // 已完成
  CANCELLED: 'CANCELLED'  // 已取消
};
```

### 4.2 状态流转图

```
                    ┌──────────┐
         ┌─────────►│  待支付   │◄──────────┐
         │          └────┬─────┘           │
         │               │                 │
    用户取消        支付成功          店主设置
         │               │                 │
         │               ▼                 │
         │          ┌──────────┐           │
         │          │  已支付   │──────────┘
         │          └────┬─────┘
         │               │
         │         店主开始制作
         │               │
         │               ▼
         │          ┌──────────┐
         │          │  制作中  │
         │          └────┬─────┘
         │               │
         │         制作完成
         │               │
         │               ▼
         │          ┌──────────┐
         └──────────│  已完成   │
                    └──────────┘
```

### 4.3 状态校验

```javascript
const StatusTransitions = {
```

## 八、性能考虑

### 8.1 优化措施

- 内存存储：订单数据存储在 Map 中，O(1) 访问
- 事件广播：状态变更实时推送，无需轮询
- 按需渲染：只在必要时刷新 TUI

### 8.2 扩展建议

- Redis 集群：多实例部署时共享状态
- WebSocket：浏览器访问支持
- 数据库：MySQL/PostgreSQL 持久化
